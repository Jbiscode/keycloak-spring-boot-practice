# 커밋 메시지 컨벤션

## 형식

```
<타입>(<범위>): <제목>

[본문]

[꼬리말]
```

### 타입 (필수)

| 타입 | 설명 |
|------|------|
| `feat` | 새로운 기능 추가 |
| `fix` | 버그 수정 |
| `refactor` | 코드 리팩토링 (기능 변경 없음) |
| `style` | 코드 포맷팅, 세미콜론 누락 등 (로직 변경 없음) |
| `test` | 테스트 코드 추가 또는 수정 |
| `docs` | 문서 수정 (README, 주석 등) |
| `chore` | 빌드 설정, 패키지 관리, 기타 잡일 |
| `ci` | CI/CD 파이프라인 설정 변경 |
| `perf` | 성능 개선 |
| `revert` | 이전 커밋 되돌리기 |

### 범위 (선택)

변경 사항이 영향을 미치는 모듈 또는 영역을 명시합니다.

예시: `auth`, `config`, `keycloak`, `docker`, `api`, `db`

### 제목 규칙

- 한국어로 작성
- 50자 이내
- 명령형으로 작성 (예: "추가", "수정", "제거")
- 끝에 마침표 없음

### 본문 (선택)

- 72자마다 줄바꿈
- 변경 이유, 변경 전후 비교 등 부가 설명 작성

### 꼬리말 (선택)

- `이슈 닫기: #123` — 관련 이슈 참조
- `관련 이슈: #123` — 닫지 않고 참조만
- `BREAKING CHANGE: <설명>` — 하위 호환성이 깨지는 변경

---

## 예시

```
feat(auth): Keycloak OAuth2 로그인 기능 추가

Keycloak을 통한 소셜 로그인 흐름을 구현했습니다.
Authorization Code Flow를 사용하며, 토큰 갱신 로직을 포함합니다.

이슈 닫기: #12
```

```
chore: .gitignore 및 커밋 컨벤션 초기 설정
```

```
fix(keycloak): 토큰 만료 시 무한 리다이렉트 문제 수정
```

```
refactor(config): 환경변수 로딩 방식 분리
```
